## 2. 本日（2026-03-22）に発生したエラーと実施した修正

### [修正済み] use_auth_token → token
- **エラー**: `pyannote.audio` の `Pipeline.from_pretrained()` 引数名の変更（ライブラリ仕様変更）
- **修正**: `use_auth_token=` を `token=` に変更

### [修正済み] ffmpeg / ffprobe のパスエラー
- **エラー**: `Couldn't find ffmpeg / ffprobe` (RuntimeWarning)
- **修正**: `AudioSegment.converter` と `AudioSegment.ffprobe` へ明示的にパス設定

### [修正済み] pydub の `codec_type` エラー
- **エラー**: OGGファイル読み込み時に `pydub` 内部の ffprobe JSON解析が `'codec_type'` キーを見つけられず失敗
- **修正**: `pydub` を廃止し、`subprocess` + `imageio_ffmpeg` バイナリ経由で直接 `ffmpeg` を呼び出して WAV に変換する方式に変更

### [修正済み] プロジェクト直下の ffmpeg.exe が壊れていた
- **エラー**: `Ticket File Not Exist` (配置した ffmpeg.exe が正規品でないライセンス付きバイナリだった)
- **修正**: `pip install imageio-ffmpeg` パッケージに同梱されている正規バイナリをコードで自動取得して使用
  ```python
  import imageio_ffmpeg
  ffmpeg_path = imageio_ffmpeg.get_ffmpeg_exe()
  ```

### [修正済み] torchaudio の `AudioDecoder is not defined` / `WinError 87`
- **エラー**: Windows上で `torchaudio.load()` が内部の `torchcodec` を使おうとして失敗
- **修正**: `torchaudio` を廃止し、`soundfile` (`sf.read()`) で NumPy 配列として直接読み込んで PyTorch テンソルに手動変換

### [修正済み] pyannote の 403 Forbidden（モデルアクセス拒否）
- **エラー**: HuggingFaceのコミュニティモデル `pyannote/speaker-diarization-community-1` へのアクセスが拒否された
- **修正**: ユーザーに当該モデルページにて利用規約への同意を促し、解決

## 3. 現在残っている問題（次回修正が必要）

### 🔴【最重要】DiarizeOutput が正しくイテレートできない
- **症状**: `[話者割り当て] エラー発生 (Type: DiarizeOutput): 'DiarizeOutput' object is not iterable`
- **影響**: 話者分離は正常に実行・完了しているが、その結果をセグメントに割り当てる処理が全て失敗し、全セグメントが `Speaker_??` になる
- **Whiskerの出力自体は正しく動いている**（文字起こしテキストは出力されている）

#### 調査済みの事実
- `pipeline()` の戻り値は `pyannote.core.Annotation` **ではなく** `DiarizeOutput` という別の型
- `Annotation` には `itertracks()` メソッドがあるが、`DiarizeOutput` にはない
- `DiarizeOutput` はイテレートもできない（`not iterable`）
- `iterrows()` も存在しない

#### 次回の調査・修正方針
1. `DiarizeOutput` 型の実際のメンバーとデータ構造を確認する
   ```python
   # テスト用コード（短い音声で実行する）
   result = pipeline(audio_dict)
   print(type(result))
   print(dir(result))
   # 属性の値を直接確認
   for attr in dir(result):
       if not attr.startswith('_'):
           try:
               print(f"  {attr} = {getattr(result, attr)}")
           except: pass
   ```
2. `DiarizeOutput` が `Annotation` を内包している可能性がある（例: `.annotation` 属性など）
3. もしくは `pipeline` を `SpeakerDiarization` ではなく `Annotation` を直接返す設定に変更できる可能性がある

## 4. 現在の `src/main.py` のアーキテクチャ

```
1. Config クラス（モデル・パラメータ設定）
2. get_initial_prompt()：initial_prompt.txt からプロンプト読み込み
3. preprocess_audio()：imageio_ffmpeg + subprocess でOGG→WAV変換 + soundfileで正規化
4. perform_diarization()：pyannote で話者分離（soundfile+torch でインメモリ読み込み）
5. assign_speaker()：話者分離結果とWhisperセグメントを照合 ← ここがバグ
6. transcribe_audio()：Whisperで文字起こし → assign_speaker で話者ラベル付与 → TXT出力
```
