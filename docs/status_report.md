# 🎙️ 文字起こしプロジェクト：現状報告と課題まとめ

## 1. 現在のシステム環境
- **ハードウェア**: NVIDIA GeForce RTX 3060 Ti (GPU活用中)
- **使用ライブラリ**: `faster-whisper`
- **使用モデル**: `large-v3`
- **OS**: Windows 10/11 (Python .venv 仮想環境)
- **解決済みの問題**:
    - `cublas64_12.dll` および `zlibwapi.dll` の認識エラー解消（環境変数 Path への動的追加による解決）。
    - GPU（CUDA）による高速化の有効化。

## 2. 現在の主な課題
- **精度の低さ**: `large-v3` を使用しているにもかかわらず、期待した変換精度が出ていない（誤字脱字、文脈の不自然さ）。
- **長尺データの処理**: 2時間の音声ファイルを処理するため、精度と速度のバランスをさらに最適化する必要がある。

## 3. 精度向上に向けた具体的アクション（今後試すべきこと）

### A. transcribe 関数のパラメータ調整
- **initial_prompt の導入**: 専門用語、固有名詞、話し方のスタイル（です・ます調など）を事前に与えて、AIに「文脈のヒント」を与える。
- **language="ja" の固定**: 自動判別による誤認を防ぎ、日本語としての一貫性を高める。
- **beam_size の調整**: 現在は 5 だが、精度重視なら 7〜10 程度に上げる（計算量は増える）。

### B. モデルの変更検討
- **large-v2 への変更**: 日本語においては `large-v3` より `v2` の方が安定しているという報告が多く、比較検証の価値あり。
- **turbo モデルの試用**: 最新の高速・高精度モデル。`large-v3` と同等の精度で速度が劇的に向上する可能性がある。

### C. 音声データの「前処理」
- **音量の正規化（Normalization）**: 小さい声を大きくし、AIの聞き取り漏れを防ぐ。
- **ノイズ除去**: 定常的な背景ノイズをカットし、声の輪郭をはっきりさせる。

### D. 文字起こし後の「AI補正」（ポストプロセス）
- **LLM（Gemini/ChatGPT等）による修正**: 文字起こしされた「荒いテキスト」を、大規模言語モデルに流し込み、「文脈に沿って正しい日本語に校正」させる。これが現在の最高精度を出すための標準的な手法。
