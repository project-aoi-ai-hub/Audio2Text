# エラーおよび修正履歴 (Error History)

## [2026-03-22] pydubおよびtorchaudio(pyannote)によるロード失敗

### 1. エラーの概要
`run.bat` を実行し、文字起こしの前処理および話者分離の段階で以下の2つのエラーが連続して発生した。
- **前処理エラー**: `[エラー] 音声前処理中にエラーが発生しました: 'codec_type'` (pydub 内部エラーにより正規化がスキップされた)
- **話者分離エラー**: `[エラー] 話者分離の処理中にエラーが発生しました: [WinError 87] パラメーターが間違っています。: '.'` (torchaudio / pyannote 読み込みによるクラッシュ)

### 2. エラーの原因、なぜ解決できていなかったのか
1. **pydub の codec_type エラー**:
   - `ffmpeg` および `ffprobe` のパスを通すことでコマンド自体の呼び出しは成功するようになったが、解析対象のファイル (`.ogg`) のフォーマット特性や ffmpeg のバージョンの組み合わせによって、`ffprobe` が出力するJSONメタデータに `codec_type` という必須キーが含まれないケースが存在する。前回は「パスが通っていないこと」のみをエラー原因と断定し、`ffprobe` の出力内容自体の解析異常まで想定できていなかった。
2. **torchaudio の [WinError 87]**:
   - 前回の修正で「ファイルを直接 pyannote に渡さず、torchaudio でメモリに展開してから渡す」という回避策をとった。しかし、裏側で `torchaudio.load()` が依然として不完全な `torchcodec`（または Windows 開発者モードに依存するライブラリ）を経由してデコードを行おうとし、そのパラメータ処理で `WinError 87` を引き起こした。私が「`torchaudio.load()` を使えばエラーを回避できる」と思い込んでおり、バックエンドを完全に明示・隔離できていなかったのが原因。

### 3. エラーの対策（今回の修正方針）
- **前処理の堅牢化 (pydub の脱 ffprobe)**:
  `pydub` にファイルを直接読み込ませるのをやめ、Python の標準モジュールである `subprocess` を使って**直接 `ffmpeg.exe` のコマンドラインを呼び出し、事前に安定した標準 WAV フォーマットに変換**する。WAV 形式であれば `pydub` は `ffprobe` を経由せずに Python ネイティブの `wave` ライブラリで安全に読み込みと正規化ができる。
- **話者分離の堅牢化 (torchaudio の完全バイパス)**:
  `torchaudio.load()` を使うのをやめ、先立ってインストールした安定の `soundfile` ライブラリを使い、**NumPy および PyTorch (torch) で手動でテンソルに変換**（バックエンドの依存を完全に断ち切る）してから pyannote に渡す。これにより、Windows特有の `torchcodec` や `torchaudio` エラーが介入する余地を物理的になくす。
