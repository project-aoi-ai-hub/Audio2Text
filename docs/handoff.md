# プロジェクト引き継ぎ用データ (Audio2Text)

## 1. 現在のプロジェクト名・目的
- **プロジェクト名**: Audio2Text
- **目的**: 音声ファイルを読み込み、テキストデータに変換して保存する。
- **目標**: 高精度なモデル（Whisper large-v3）を、ローカルPCのGPU（CUDA）を活用して高速に動作させる。

## 2. 決定事項・確定した仕様
- **音声認識バックエンド**: `faster-whisper` を採用。
- **実行モデル**: デフォルトで `large-v3` を使用。
- **実行環境**: NVIDIA GPU(CUDA)環境を優先して利用。DLL等の複雑なシステムインストールを避け、Pythonパッケージ (`nvidia-cublas-cu12` 他) を利用して環境を自己完結させる「アプローチ A」を採用。
- **出力仕様**: `[音声ファイル名]_transcription.txt` という名前で、元ファイルと同じフォルダに保存する。

## 3. 現在のステータス
- **環境構築**: 完了。`.venv` 下に必要なライブラリがインストール済み。
- **CUDA動作対応**: 完了。`src/main.py`内にてライブラリPATHを動的に追加することで、Windows環境で見つかりがちな DLL 不足エラーを回避済み。
- **実行インターフェース**: `run.bat` (実行用) および `setup.bat` (構築用) を作成。ダブルクリックで即座に実行可能な状態。

## 4. 未解決の課題・次にやるべきこと
- **出力先の変更**: 現在、元ファイルと同じ場所に生成されるが、専用の `output` フォルダなどに集約する機能の検討。
- **詳細な読み起こし設定**: タイムスタンプの有無や、話者分離機能（Diarization）の要否。
- **一括処理**: フォルダ内の全ての音声ファイルをまとめて変換する機能の実装。

## 5. 重要な技術スタックや前提条件
- **Python 3.10+** (仮想環境 venv 推奨)
- **faster-whisper**: CTranslate2ベースの高速Whisper実装。
- **NVIDIA GPU (CUDA 12系)**: 高速化に必要。未検出時はCPUへ自動フォールバック。
- **Windows特有の対応**: `os.add_dll_directory` を使用して、pipでインストールしたCUDAライブラリを明示的に読み込む必要がある。
